# Coord Directory Map

Authoritative orientation for the `coord/` repo. Read this before adding,
moving, or deleting anything here. The deep restructure proposal is archived at
[`docs/archive/COORD_REPO_RESTRUCTURE_PROPOSAL.md`](./docs/archive/COORD_REPO_RESTRUCTURE_PROPOSAL.md);
this file documents the layout as it exists today and the rules that keep it stable.

> Template note: this is the canonical template copy. Downstream projects
> inherit it via sync. Project-specific document names below are illustrative
> examples — substitute your project's own product/domain specs.

## The three state buckets

`coord/` mixes three kinds of state. Knowing which bucket a file is in tells
you whether you may edit it, whether `git` history matters, and whether a
`reset`/`stash` can destroy it.

### 1. Canonical tracked — source of truth, edit deliberately

History matters; rollback is meaningful. Edit via the governed flow.

- Governance: `GOVERNANCE.md`, `GOVERNANCE_ARCHITECTURE.md`, `AGENTS.md`,
  `VERB_CONTRACT.md`, `README.md`
- Agent model: `AGENT_CAPABILITY_REGISTRY.md`, `AGENT_PATHS.md`,
  `AGENT_STARTUP_CHECKLIST.md`
- Product/domain specs: `product/**` and project-specific root specs
  (each downstream project owns these — e.g. its URS, domain model,
  config-inheritance model, architecture plan, reference slice)
- Board source: `board/tasks.json` (lifecycle status/locks owned by `gov` —
  new backlog rows may be hand-authored, then `node board/board.js sync`).
  `tasks.json` stays canonical; plan-record shards are no longer canonical
  (see bucket 3 — they moved to `.runtime/plans/`)
- Prompts: `prompts/**`
- Design notes: `docs/**`
- Mapping/audit references kept at root by convention

### 2. Derived / generated — NEVER hand-edit

Produced by `node coord/board/board.js sync` from bucket 1. Hand-edits are
overwritten and cause drift. Commit only alongside their canonical source.

- `rendered/TASKS.md`, `rendered/PROMPT_INDEX.md` — the tracked
  derived source views (derived-but-reviewed; commit with their source)
- `TASKS.md`, `PROMPT_INDEX.md` — root-level compat mirrors of the
  `rendered/*` views. Still written on every `board.js sync`
  (`writeCompatibilityCopy`) for consumers, but **gitignored** (C6-P3):
  no longer tracked. Do not re-add them to git.
- `PLAN.md` (rendered plan ledger — empty is normal; still tracked)
- `TEST_MATURITY.md` (regenerated by the test-strategy workflow)
- `artifacts/**` run outputs (gate / reference-acceptance latest snapshots)

### 3. `.runtime/` — split buckets (ENT-001): durable-tracked vs ephemeral-ignored

`coord/.runtime/` is **ignored by default**, but the durable governance-evidence
buckets are **un-ignored (tracked)** so a fresh clone can validate its own board
and run the governance suite without missing-evidence/freshness failures. The
split is enforced by anchored `.gitignore` negations in BOTH the root
`.gitignore` and `coord/.gitignore` (the deeper file is consulted for paths
under `coord/`, so the negation must live in both). Use `.runtime/*` (not
`.runtime/`) so git descends and the `!` negations re-include specific paths.

**Tracked (durable evidence — committed by the post-finalize sync):**
- `.runtime/governance-events.ndjson` — the governance journal (source-of-record
  event log).
- `.runtime/governance-latest-snapshot.json` + `.runtime/governance-snapshots/`
  — snapshot artifacts.
- `.runtime/plans/*.json` — canonical plan records (runtime-owned; written by
  `gov`/`governance.js`). A **temporary compatibility reader** still resolves
  shards from the legacy tracked location `board/plans/*.json` when no
  `.runtime/plans/` copy exists; `board.js` unions both dirs for validation
  (runtime wins on collision).

**Ephemeral (still gitignored — rebuildable / per-process, never durable):**
- `.runtime/locks/`, `.runtime/locks/*.lock` — live ticket locks.
- `.runtime/agent_sessions.json`, `.runtime/session-instances.json`,
  `.runtime/session-threads/`, `.session-thread-*` — session state.
- `.runtime/agents.json` — agent registry mirror.
- `.runtime/gate-procs/` — gate process-orphan pidfiles.
- `.runtime/governance.lock/` — journal append lock.
- Other scratch (`tmp*`, `*.scratch`). Plus `.worktrees/`, `memory/`.

The tracked journal/plans/snapshots join `canonicalSyncablePaths()` (see
`scripts/state-io.js`), so every lifecycle boundary (`finalize`/`land`/
`mark-done`) durably commits them in the SAME scope-limited sync commit as the
board. Because the journal and board are committed together, the doctor
"tracked board state predates the governance journal" freshness advisory no
longer fires after a clean finalize.

> **Push-on-finalize (opt-in):** the post-finalize sync commits locally by
> default. To also push it to the coord remote, pass `--push-after-sync` on a
> terminal verb (`finalize`/`land`/`mark-done`/`close`/`finish`) or set
> `COORD_PUSH_ON_FINALIZE=1`. It does a plain (non-force) `git push`; it is
> never the default and skips quietly if no upstream is configured.

> **Migration (existing deployments whose journal/plans were gitignored):**
> after adopting this `.gitignore` change, the previously-ignored
> `.runtime/governance-events.ndjson`, `.runtime/governance-snapshots/`, and
> `.runtime/plans/*.json` become trackable. The **first** `gov sync --commit`
> (or any finalize/land, which auto-syncs) stages and commits the accumulated
> journal + plan + snapshot history as a one-time **migration baseline** — an
> intentionally large commit; that is the point. No data migration or tooling
> is required: just run a finalize/sync and review the baseline commit. Locks
> and sessions are never picked up, so live state is unaffected.

## Template-managed paths — DO NOT restructure downstream

These paths are owned by this template and tracked for exact-match drift by
`TEMPLATE_SYNC_MANIFEST.json`. Moving, renaming, or refactoring any of them
in a downstream project breaks drift detection. Changes are
**template-first**: make them here in `coord-template`, then sync down.

```
AGENT_CAPABILITY_REGISTRY.md   AGENT_PATHS.md   GOVERNANCE_ARCHITECTURE.md
VERB_CONTRACT.md
board/board.js   board/board.test.js   board/plan.schema.json   board/tasks.schema.json
scripts/gov   scripts/governance.js   scripts/governance.test.js
scripts/governance-mcp.js   scripts/check-template-sync.sh
scripts/check-template-sync.test.js   scripts/preflight.sh
docs/AGENT_ID_COLLISION_RECOVERY_POLICY.md   docs/IDENTITY_RUNTIME_EXTRACT.md
docs/provider-thread-id-sources.md
```

The exact tracked set is whatever `TEMPLATE_SYNC_MANIFEST.json` lists for the
project; the list above is the template-canonical baseline.

`paths.js` resolves the repo layout through project-owned
`project.config.js`: moving `board/`, `active/`, `prompts/`, `.runtime/`
breaks `governance.js` and `board.js`. A downstream project defines repo
codes, integration branches, origins, and legacy aliases in
`project.config.js`; `paths.js` is engine-managed and must not be hand-edited
for project layout.

## Directory index

| Path | Bucket | Purpose |
|------|--------|---------|
| `active/` | canonical (low-churn archive) | One markdown spec per ticket; mostly post-land archives |
| `artifacts/` | derived | Gate + reference-acceptance run outputs |
| `board/` | canonical | `tasks.json` (canonical) + `board.js` + schemas. `plans/*.json` shards moved to `.runtime/plans/` (legacy copies read-only via compat reader) |
| `config/` | canonical | Coordination config |
| `docs/` | canonical | Design notes, identity/collision policies, plans |
| `locks/` | ephemeral | Legacy lock dir (gitignored; live locks in `.runtime/locks/`) |
| `memory/` | ephemeral | Agent memory bin (gitignored) |
| `product/` | canonical | Product + domain specs |
| `prompts/` | canonical | Role prompts (planner/implementer/reviewer) |
| `reference-acceptance/` | canonical | Reference acceptance harness inputs |
| `rendered/` | derived | Generated `TASKS.md` / `PROMPT_INDEX.md` copies |
| `scripts/` | canonical (mostly template-managed) | Governance CLI + MCP |
| `tools/` | canonical | Supporting tools |
| `.runtime/` | split (ENT-001) | **Tracked:** journal (`governance-events.ndjson`), snapshots, `plans/`. **Ignored:** locks, sessions, `agents.json`, `gate-procs/`, scratch. See bucket 3. |
| `.worktrees/` | ephemeral | Per-agent/ticket workspaces (gitignored; repo X uses empty dirs) |

## Deferred (template-first, not done downstream)

The full canonical/derived/ephemeral split — relocating `board/plans/*.json`
to a runtime path, retracking generated views as generate-only, narrowing
tracked `artifacts/**`, and ignoring transient `active/` notes — is
intentionally **not** done in downstream projects. It alters template-managed
and path-resolution surfaces and must flow through `coord-template`. See
`docs/archive/COORD_REPO_RESTRUCTURE_PROPOSAL.md`.
